# CityWater Data Model

Canonical reference for the database schema. Source of truth is `backend/app/models/` (SQLAlchemy declarative `Mapped[...]`) plus `backend/migrations/versions/` (Alembic). This doc is a high-level overview for onboarding.

## Conventions (enforced repo-wide; see CLAUDE.md)

- Every table: `id BIGSERIAL PK`, `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`, `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()` (exception: append-only event tables ‚Äî `audit_log` ‚Äî use `occurred_at` and omit `updated_at`).
- Every domain table: `tenant_id BIGINT NOT NULL REFERENCES tenant(id)` and the session-level filter (`app/services/tenancy.py`) ensures every ORM query is tenant-scoped.
- Every FK and index is named (no autogenerated `ix_‚Ä¶_fkey`).
- Spatial columns: `GEOMETRY(<type>, 4326)` with a named GIST index.
- JSONB columns that are filtered have a GIN index (typically `jsonb_path_ops`).
- Soft delete via `deleted_at TIMESTAMPTZ NULL` on mutable domain entities; junction tables and event/audit tables are hard-delete only.
- All natural keys (`asset_uid`, `wo_number`, `sr_number`, `inspection_number`, role/code, etc.) are unique scoped to `(tenant_id, ...)` ‚Äî never globally.

## Domain entities

### Tenancy + Identity

- `tenant` ‚Äî top-level isolation boundary
- `user`, `role`, `user_role` ‚Äî user identity and role assignment (per-tenant role rows)
- `invitation` ‚Äî admin-initiated user provisioning with single-use tokens
- `audit_log` ‚Äî append-only event stream; not tenant-scoped via the listener (queries must scope explicitly)

### Catalog (global, shared across tenants)

- `asset_class` ‚Äî the only global catalog. Defines geometry type + JSON Schema for `asset.attrs`. Edits are gated by `ALLOW_ASSET_CLASS_EDITS` env because the table has no `tenant_id`.
- `pacp_code` ‚Äî NASSCO PACP defect code reference for CCTV inspections.

### Operational entities

- `asset` ‚Äî Point/LineString/Polygon assets (hydrants, valves, mains, manholes, catch basins, lift stations, ‚Ä¶). `attrs JSONB` validated against `asset_class.attribute_schema`.
- `service_area` ‚Äî polygon overlays (maintenance districts, water systems, sewer systems, storm systems). `kind` discriminator + GIST index for spatial intersect.
- `work_order` ‚Äî the core unit of work. `category`, `priority`, `status` enums (extended by 0026). Soft-delete. JSONB `attrs` and `task_data`. Carries optional `template_id`, `service_request_id`, `schedule_id` foreign keys.
  - `work_order_asset` ‚Äî M:N: WO ‚Üî asset with `role`, `sequence`, per-stop `completed_at`. Carries denorm `tenant_id` (added in 0029).
  - `work_order_task` ‚Äî checklist items
  - `work_order_time_log` ‚Äî labour
  - `work_order_material` ‚Äî materials consumed
  - `work_order_attachment` ‚Äî uploaded files (S3 key + EXIF)
- `wo_template` ‚Äî reusable WO blueprints (used by recurring schedules)
- `inspection` ‚Äî typed inspections (hydrant flow, valve exercise, manhole, catch basin, lift station round, CCTV/PACP). Per-kind payload in `data JSONB`.
- `service_request` ‚Äî citizen/intake-driven requests; can be dispatched as a WO. `attrs JSONB`, polymorphic `task_definition_id`.
- `comment` ‚Äî polymorphic comments on WO/SR/Inspection
- `entity_link` ‚Äî polymorphic links between WO/SR/Inspection (e.g. "this inspection produced this WO")
- `schedule` ‚Äî RRULE-driven (RFC 5545) recurring spawner for WO/inspection
- `task_definition` ‚Äî tenant-customisable task catalog with versioning (draft/active/archived), prefill, validation, completion rules
- `crew`, `crew_member` ‚Äî operational crew lists for assignment
- `geocode_queue` ‚Äî async geocoding work queue

## Migrations

Linear chain `0001_postgis_extension` ‚Üí `0029_audit_hardening`. Every revision has a matching `downgrade()`. Never edit a migration after it ships ‚Äî add a new one. Two-phase rollouts for destructive column drops:

1. Deploy code that ignores the column (read + write paths bypass it).
2. Deploy migration that drops the column.

## Spatial

PostGIS 3.4+. SRID 4326 everywhere. Spatial work uses `ST_*` functions in SQL, never haversine in Python. MVT vector tiles served via `assets_mvt(tenant_id, z, x, y)` PL/pgSQL function (see migration 0007).
